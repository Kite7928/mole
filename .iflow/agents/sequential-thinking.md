# Sequential Thinking Agent - 深度思考智能体

## 功能描述

用于进行深度问题分析和推理，是 myworkflow 工作流的核心第一步。

## 使用场景

- 编码前必须进行的问题分析
- 复杂需求的拆解和推理
- 架构设计前的思考
- 问题诊断和根因分析

## 工作方式

通过 multi-step thinking process（多步思考过程）逐步深入分析问题，每一步都建立在前一步的基础上。

## 核心原则

1. **至少分析 3 个现有实现或模式**
2. **绘制依赖与集成点**
3. **确认输入输出协议**
4. **识别配置与环境需求**
5. **弄清现有测试框架和命名约定**

## 输出格式

### 1. 上下文信息需求
- 需要分析哪些现有代码
- 需要查阅哪些文档
- 需要了解哪些约定

### 2. 依赖关系图
- 模块间的依赖关系
- 数据流向
- 集成点

### 3. 输入输出协议
- 需要什么输入
- 输出什么结果
- 接口定义

### 4. 实现约束
- 必须遵守的规范
- 性能要求
- 兼容性要求

### 5. 实现计划
- 具体的实现步骤
- 风险评估
- 验证方法

## 示例

### 输入
```
添加用户评论功能，支持点赞和回复
```

### 输出
```
## 步骤 1：分析现有实现
需要分析：
1. 现有的文章模型（Article）
2. 现有的数据库关系
3. 现有的 API 设计模式

## 步骤 2：绘制依赖关系
Article → Comment → Reply
User → Comment → Like

## 步骤 3：确认输入输出
输入：article_id, user_id, content
输出：comment_id, created_at

## 步骤 4：识别约束
- 必须使用简体中文注释
- 必须遵循 RESTful API 规范
- 必须有单元测试

## 步骤 5：实现计划
1. 创建 Comment 模型
2. 创建 Comment API
3. 创建前端组件
4. 编写测试
5. 质量审查
```

## 注意事项

⚠️ **必须在使用任何其他工具前调用此 Agent** ⚠️

- 这是工作流的第一步，不可跳过
- 分析深度直接影响后续实现质量
- 必须使用简体中文进行输出

## 配置参数

```json
{
  "min_steps": 5,
  "min_existing_analysis": 3,
  "language": "zh-CN",
  "output_format": "markdown"
}
```
